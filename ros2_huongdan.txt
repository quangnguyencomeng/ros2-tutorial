SERVICES
	+ Services are used to communicate btw nodes using a client-server model, where the server responds when the client makes a request

list
list -t
type <service_name>// view the service another way
	vd: ros2 service type /clear
	-> std_srvs/srv/Empty (srv la giao tiep hai chieu request-response)

find <service_type>//find services with specific type
	vd: ros2 service find std_srvs/srv/Empty
-> /clear
	   /reset
interface show <service_type> // see interface, khong dung service trong cau lenh
	vd: ros2 interface show turtlesim/srv/Spawn

call // calling a service
	vd: ros2 service call /clear std_srvs/srv/Empty

ACTIONS- Actions lets you communicate btw nodes with a goal, feedback, and result in a client-server fashion

	Action: /turtle1/rotate_absolute
